# WxO における MCP サーバーの接続方式

MCP（Model Context Protocol）は、AI エージェントが外部ツールやデータソースに接続するための通信規格です。
IBM watsonx Orchestrate（WxO）では、MCP サーバーの接続方式を2種類定義しています。

参考: [WxO 公式ドキュメント — MCP サーバー](https://www.ibm.com/docs/ja/watsonx/watson-orchestrate/base?topic=tools-mcp-servers)

---

## リモート MCP サーバー（StreamableHTTP / SSE）

MCP サーバーを**インターネット上の公開サーバーに置く**方式です。

MCP サーバーを Railway、Render、AWS などのクラウドサービスにデプロイし、公開 URL で提供します。
WxO エージェントは HTTPS（StreamableHTTP）または SSE でその URL に接続します。

この方式の特徴は、クラウド上の WxO エージェントからも接続できることです。
ただし、MCP サーバーをホストするためのサーバーインフラを自分で用意・運用する必要があります。
デプロイ先の選定、セキュリティ設定、ドメインや証明書の管理など、インフラの運用コストがかかります。

---

## ローカル MCP サーバー（STDIO）

MCP サーバーを**WxO クラウド内でコマンドとして起動する**方式です。今回採用した方式です。

WxO の MCP Toolkit 設定ファイルに `command:` としてコマンドを書くと、WxO がそのコマンドを自身のクラウド環境（OpenShift コンテナ）上で起動します。WxO エージェントと MCP サーバーは STDIO（標準入出力）でやり取りします。

> **「ローカル」という言葉について**
> WxO ドキュメントの「ローカル MCP サーバー」は、ユーザーの手元の PC を指しません。WxO が自身のクラウド環境内でコマンドとして起動・管理する MCP サーバーのことです。

この方式の大きな特徴は、ユーザーが MCP サーバーを動かすための環境を一切用意しなくてよいことです。
手元の PC に Node.js をインストールする必要はありません。公開 URL を持つサーバーを用意する必要もありません。
データベースへの接続情報（DATABASE_URL など）は WxO の Connection 機能でセキュアに管理され、MCP サーバープロセスの環境変数として自動的に注入されます。

---

## 2つの方式の比較

| | リモート MCP サーバー | ローカル MCP サーバー |
|---|---|---|
| MCP サーバーの実行場所 | 自前のクラウドサーバー（公開 URL） | WxO クラウド内（コマンド起動） |
| 接続方式 | StreamableHTTP / SSE | STDIO |
| 公開サーバーの用意・運用 | 必要 | 不要 |
| PC への Node.js 等のインストール | 不要 | 不要 |
| 設定の手軽さ | インフラ知識が必要 | YAML に `command:` を書くだけ |

---

## まとめ

リモート方式は MCP サーバーを自前で公開・運用する必要があります。
ローカル方式（今回）は PC も公開サーバーも不要で、WxO エージェントと MCP を最も手軽に組み合わせられる方法です。
